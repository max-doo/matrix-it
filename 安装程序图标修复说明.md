# 安装程序图标修复说明

## 问题描述

打包完成后，安装程序（MSI 和 NSIS）仍然显示默认图标，而非应用自定义图标。

## 解决方案

### 1. WiX MSI 安装程序图标配置

#### 修改内容
在 `src-tauri/wix/fragments/ui-custom.wxs` 中添加产品图标定义：

```xml
<?xml version="1.0" encoding="utf-8"?>
<Wix xmlns="http://schemas.microsoft.com/wix/2006/wi">
  <Fragment>
    <!-- 产品图标定义 - 用于安装程序和添加/删除程序 -->
    <Icon Id="ProductIcon" SourceFile="icons\icon.ico" />
    
    <!-- 桌面快捷方式组件 -->
    <DirectoryRef Id="DesktopFolder">
      <Component Id="DesktopShortcut" Guid="...">
        <Shortcut Id="DesktopShortcut"
                  Name="MatrixIt"
                  Icon="ProductIcon" />  <!-- 引用产品图标 -->
        ...
      </Component>
    </DirectoryRef>
  </Fragment>
</Wix>
```

#### 作用
- **MSI 文件图标**：控制面板"程序和功能"中显示的图标
- **桌面快捷方式图标**：使用 ProductIcon
- **安装向导图标**：（部分支持，取决于 WiX 版本）

### 2. NSIS 安装程序图标配置

#### 修改内容
在 `src-tauri/tauri.conf.json` 的 `bundle.windows.nsis` 中添加：

```json
{
  "bundle": {
    "windows": {
      "nsis": {
        "displayLanguageSelector": false,
        "languages": ["SimpChinese"],
        "installerIcon": "icons/icon.ico"  ← 新增
      }
    }
  }
}
```

#### 作用
- **NSIS 安装程序图标**（`MatrixIt_1.0.0_x64-setup.exe`）：资源管理器中的文件图标
- **安装向导窗口图标**：安装过程中窗口标题栏显示的图标

## 配置文件完整内容

### tauri.conf.json
```json
{
  "bundle": {
    "icon": [
      "icons/icon.ico",    // 应用程序图标
      "icons/icon.png"
    ],
    "windows": {
      "wix": {
        "language": {
          "zh-CN": {
            "localePath": "wix/locales/zh-CN.wxl"
          }
        },
        "fragmentPaths": [
          "wix/fragments/ui-custom.wxs"    // 包含 ProductIcon 定义
        ],
        "componentRefs": [
          "DesktopShortcut"
        ]
      },
      "nsis": {
        "displayLanguageSelector": false,
        "languages": ["SimpChinese"],
        "installerIcon": "icons/icon.ico"  // NSIS 安装程序图标
      }
    }
  }
}
```

## 重新打包

配置修改后，需要重新打包：

```powershell
cd src-tauri
cargo tauri build
```

## 验证方法

### MSI 安装程序
1. 找到文件：`src-tauri/target/release/bundle/msi/MatrixIt_1.0.0_x64_zh-CN.msi`
2. **验证文件图标**：
   - 在资源管理器中查看 `.msi` 文件图标
   - 刷新资源管理器（F5）确保图标缓存更新
3. **验证快捷方式图标**：
   - 安装应用后检查桌面快捷方式图标
   - 检查开始菜单图标

### NSIS 安装程序
1. 找到文件：`src-tauri/target/release/bundle/nsis/MatrixIt_1.0.0_x64-setup.exe`
2. **验证安装程序图标**：
   - 在资源管理器中查看 `.exe` 文件图标
   - 应该显示为 MatrixIt 应用图标
3. **验证安装向导图标**：
   - 双击运行安装程序
   - 检查窗口标题栏左上角图标

## 图标应用位置总结

配置完成后，自定义图标会应用到以下位置：

### 应用程序本身
- ✅ `MatrixIt.exe` - 主程序图标（`bundle.icon`）
- ✅ 任务栏图标
- ✅ Alt+Tab 切换器图标

### 安装程序
- ✅ **MSI 文件图标** - 资源管理器中的 `.msi` 文件
- ✅ **NSIS 文件图标** - 资源管理器中的 `.exe` 文件（新增配置）
- ✅ **NSIS 安装向导图标** - 安装窗口标题栏（新增配置）

### 安装后
- ✅ **桌面快捷方式图标** - WiX ProductIcon 引用
- ✅ **开始菜单图标** - Tauri 默认
- ✅ **程序和功能图标** - 控制面板列表（WiX ProductIcon）

## 图标文件要求

为确保最佳显示效果，`icons/icon.ico` 应包含多种尺寸：

```
icon.ico（推荐层级）:
- 16x16 (小图标)
- 32x32 (任务栏)
- 48x48 (资源管理器)
- 64x64 (高DPI)
- 128x128
- 256x256 (高分辨率)
```

### 检查图标文件
```powershell
# 查看当前图标文件大小
Get-Item "src-tauri\icons\icon.ico" | Select-Object Name, Length
```

如果图标文件较小（< 10KB），可能缺少高分辨率层级，建议重新生成：

```powershell
npm run tauri icon path/to/source-icon.png
```

## 故障排除

### 问题 1：MSI 文件图标未显示
**原因**：WiX 默认模板可能未使用 Fragment 中定义的图标

**解决方案**：
1. 确认 `ui-custom.wxs` 中的 Icon 定义正确
2. 重新打包
3. 清除 Windows 图标缓存（见下文）

### 问题 2：NSIS 图标未显示
**原因**：`installerIcon` 路径错误或图标格式问题

**解决方案**：
1. 确认路径为 `icons/icon.ico`（相对于 `src-tauri/`）
2. 确认图标文件包含多种尺寸
3. 重新打包

### 清除 Windows 图标缓存
```powershell
# 管理员权限运行
taskkill /f /im explorer.exe
del /a /q "%LocalAppData%\IconCache.db"
del /a /q "%LocalAppData%\Microsoft\Windows\Explorer\iconcache_*.db"
start explorer.exe
```
