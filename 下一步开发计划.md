# 开发pdf阅读器（下一期开发）

## 难度评估（用库 vs 自己从零写）
- **从零基于 pdf.js 自己写一个阅读器**：能做，但工作量明显偏大。
  - 仅“渲染 + 缩放/翻页/滚动”：中等
  - “可选中文本/复制”：需要 TextLayer，处理缩放/滚动对齐、性能与选择范围映射
  - “做标注（高亮/批注/涂鸦）+ 撤销/重做 + 保存/恢复”：会迅速变成一套完整编辑器（工作量大）
- **用现成库做**：难度会降一个量级。
  - Context7 里有一个更贴合你需求的开源方案：**EmbedPDF Viewer**（/embedpdf/embed-pdf-viewer）
  - 它提供：文本选择/复制插件（selection）、标注插件（annotation：highlight/ink 等）、以及事件监听、导入/导出标注等能力（均有示例）。

结论：
- “能复制文字 + 做基础标注（高亮/涂鸦/批注）并保存到应用数据里”——**用 EmbedPDF 做，难度中等，可控**。
- “把标注真正写回 PDF 文件（生成带标注的新 PDF）”——难度会上升一档（需要额外 PDF 写入库和坐标/页面变换处理）。

## 推荐方案（更优）
用 **EmbedPDF Viewer + Selection + Annotation 插件**，在 MatrixIt 内部新开一个“阅读器窗口/抽屉”展示 PDF，而不是依赖系统默认 PDF 关联程序。

### Phase 1：先把阅读器跑起来（可选中复制）
- 前端新增依赖（按 Context7 文档）：
  - `@embedpdf/core`、`@embedpdf/engines`、`@embedpdf/plugin-loader`、`@embedpdf/plugin-viewport`、`@embedpdf/plugin-scroll`、`@embedpdf/plugin-render`
- 新增一个 `PdfReader` React 组件：
  - 使用 `usePdfiumEngine()` 初始化引擎
  - 用 Loader 插件加载 PDF（初版先支持 URL；随后支持本地 file:// 或通过 Tauri command 获取可访问的 URL）
  - RenderLayer + Scroller 渲染页面
- 在 UI 上提供一个“在应用内阅读”入口（可替换/并列原有“阅读原文”按钮）。

### Phase 2：加入“复制文字”与“标注”能力
- 追加安装并注册插件（Context7 有明确安装与注册示例）：
  - selection：`@embedpdf/plugin-selection`（提供 copyToClipboard 能力）
  - annotation：`@embedpdf/plugin-annotation`，以及依赖 `@embedpdf/plugin-interaction-manager`、可选 `@embedpdf/plugin-history`
- 增加简单工具栏：复制、选择模式、高亮、画笔、撤销/重做。

### Phase 3：标注持久化与回填
- 监听 annotation 事件（create/update/delete），将标注数据保存到本地（建议按 `item_key` 存 JSON）。
- 打开同一篇文献时自动 import 之前保存的标注。

### Phase 4（可选）：导出带标注的 PDF
- 若你希望“真正写回 PDF”，需要额外引入 PDF 写入/重打包方案（工作量更大）；可先把标注导出成 JSON/单独文件满足主要需求。

## 验证标准
- 打开任意 PDF：渲染稳定、滚动/缩放正常
- 鼠标选中文本：可以复制到剪贴板
- 创建高亮/涂鸦/批注：能撤销/重做，关闭后再打开仍能恢复

如果你同意这条路线，我会优先按 Phase 1+2 落地，让“应用内阅读 + 复制 + 基础标注”先可用，再补 Phase 3 的持久化。
