## 结论（结合最佳实践）
- Ant Design Table 的 `columns.width` 支持 `string | number`，官方“可调整列宽”示例通常以**像素宽度（number）**为主（拖动 delta 天然是 px），更适合“列数量/可见列动态变化”的场景。
- 你现在的**百分比列宽 + mousemove 频繁 setState**，在列增减时会导致宽度比例失真；拖动时又会触发表格整树重渲染与频繁布局，造成卡顿。

## 目标
- 拖动列宽时显著更流畅（避免每个鼠标事件触发整表重渲染）。
- 列数量变化/显隐变化时列宽保持“按列 key 记忆”，不会因归一化百分比而整体漂移。
- 保留：最小列宽、邻列挤压逻辑、localStorage 持久化（按 view 分开）。

## 方案（推荐）
### 1）宽度单位：从百分比改为像素（px）
- 本地存储从 `Record<key, percent>` 改为 `Record<key, px>`。
- 初始化时：
  - 已存储的 key 直接复用 px；新增 key 用“minWidth + 权重分配 extra”生成 px。
  - 若总宽度小于容器宽度，将 extra 主要分配给 title/长文本列，使默认观感更好。

### 2）性能：拖动过程不再 setState，改用 CSS 变量直接驱动
- 给容器写 CSS 变量：`--matrixit-col-<key>: 180px`。
- `columns.width` 固定写成 `var(--matrixit-col-<key>)`（列定义不再依赖实时宽度）。
- 拖动时只做两件事：
  - 更新 `columnWidthsRef.current`（px 记录）
  - `containerRef.current.style.setProperty(...)` 更新 CSS 变量
- 松开鼠标时：一次性写入 localStorage（必要时再触发一次 setState 用于同步依赖）。

### 3）交互：改用 Pointer Events
- `pointerdown/pointermove/pointerup` + `setPointerCapture`，减少丢事件/跨平台差异。

## 实施步骤（只改现有文件，不新增文件）
1. 在 `LiteratureTable.tsx` 中将 `columnPercents` 体系替换为 `columnWidthsPx`（ref 为主，state 仅用于初始化/触发少量更新）。
2. 将列定义里的 `width` 改为 CSS 变量引用，移除 `columns` 对实时宽度 state 的依赖，避免拖动时重建列数组。
3. 将拖动计算从 percent 改为 px：
   - `dx = evt.clientX - startX`
   - `nextCurrent = clamp(startWidth + dx, minWidth)`
   - `neighbor` 同步减少并 clamp；必要时回弹 current。
4. 初始化/恢复逻辑：
   - 合并 visibleKeys 与存储值；对缺失列用权重生成默认 px
