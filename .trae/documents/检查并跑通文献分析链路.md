## 现状梳理
1) 前端点击“开始分析”会调用 `startAnalysis()`，经 `invoke('start_analysis', { item_keys, on_event })` 走 Tauri IPC。
2) Rust `start_analysis` 会 spawn sidecar：`matrixit-sidecar analyze <item_keys_json>`，解析 stdout 的逐行 JSON 事件并经 Channel 回传前端。
3) Python sidecar `analyze()`：定位 PDF→提取文本→调用 LLM（chat/completions 或 responses+PDF）→解析为 dict→把字段写回 item JSON→`upsert_item` 落 SQLite，并 `export_json` 更新 `data/literature.json`。

## 关键风险点（影响“看起来跑通/实际没写库”）
- 前端 `startAnalysis()` 的 RPC 封装当前是“无条件 catch 后伪造事件流”，即使在 Tauri 运行、invoke 失败也会显示成功，但不会真正写库。
- Tauri devUrl 固定 `http://127.0.0.1:5174`，而当前你本地 dev server 端口可能不一致会导致桌面壳加载失败/不是 Tauri runtime。
- 机密信息风险：你提供的 key 存在 `api key.md`，且 UI 保存配置会把 `llm.api_key` 写入 `config/config.json`（可提交文件），存在误提交风险。

## 执行方案（会改代码/配置以保证真实链路可观测、且不泄露密钥）
### 1) 让“失败可见”，禁止 Tauri 下伪造成功
- 修改 `frontend/src/lib/backend.ts` 的 `startAnalysis`：
  - 若 `isTauriRuntime()` 为 true：invoke 失败就抛错（让 UI 提示“启动分析失败”）。
  - 仅在非 Tauri runtime（纯浏览器开发/演示）才继续使用“伪造事件流”兜底。

### 2) UI 在分析完成后自动刷新数据（确保字段值可见）
- 修改 `frontend/src/ui/App.tsx`：
  - 在事件流收到 `AllDone` 时自动触发一次 `handleRefresh()`（或只刷新涉及的条目），从 SQLite/literature.json 重新拉取 items，确保提取到的字段值展示出来。

### 3) 安全地注入可用 LLM 配置（不把 key 写进可提交文件）
- 使用 `config/config.local.json` 存放 `llm.api_key/base_url/model`（该文件已在 `.gitignore` 忽略），sidecar 会自动合并覆盖。
- 同时处理 `api key.md`：
  - 该文件包含明文密钥，计划删除并加入 `.gitignore`（或至少加入 ignore），并建议你在云端控制台轮换该 key。
- 如你希望“在设置页填写 key 也不会落到 config.json”：我可以进一步改造 `src-tauri/src/main.rs` 的 `save_config`，把 `llm.api_key` 自动写入 `config/config.local.json`，而 `config/config.json` 内保持空/占位。

### 4) 启动并验证真实端到端链路
- 启动前端 dev server 到 5174（与 `src-tauri/tauri.conf.json` 的 devUrl 一致）。
- 启动 Tauri dev（桌面壳），确认前端处于 Tauri runtime（不走 mock）。
- 在 UI 选中一条有 PDF 的文献，点击“开始分析”。
- 验证点：
  - 前端能收到真实 `Started/Finished/Failed/AllDone`（非伪造）。
  - sidecar stdout 无 `LLM_CONFIG_MISSING`、`PDF_NOT_FOUND`、`LLM_HTTP_ERROR` 等错误。
  - 分析完成后 SQLite 的该 item JSON 内出现解析字段（如 `methods/logic/...`），并且 `data/literature.json` 已更新。

## 验收标准（你能直观看到）
- 点击按钮后，若 invoke 或 sidecar 出错，UI 明确报错而不是“假成功”。
- 分析完成后无需手动刷新，表格/详情能显示提取字段值。
- SQLite 与 `data/literature.json` 均持久化了提取结果。
