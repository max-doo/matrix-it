## 回答：本地改 name 后，能不能同步修改飞书字段名？

* 能：飞书多维表格提供“更新字段”接口，可以修改 field\_name（也可带 type 等全量更新）。来源：飞书开放平台《更新字段》（PUT /bitable/v1/apps/:app\_token/tables/:table\_id/fields/:field\_id，2025-01-09）。

* 但有一个前提：要“可靠地改名”，必须先知道要改的是哪一列（field\_id）。

  * 仅靠“新 name”是找不到旧列的：你改完 name 后，系统不知道旧列名是什么；如果只按 name 识别，会变成“找不到→新建一列”，而不是“把旧列改名”。

## 因此：要同时满足“统一用 name”+“允许改 name 自动改云端列名”，需要一个最小的“内部稳定标识”

* 这不需要你维护两套映射配置（不增加 UI 配置复杂度），但需要系统内部记住：json\_key → 飞书 field\_id（以及上次同步的列名）。

* 最适合存放位置：config/config.local.json（本地文件，不提交），用于保存：

  * feishu.schema.fields: { \<json\_key>: { field\_id: "fld...", last\_name: "..." } }
